MQ - Using the MQInput, MQGet and MQOutput nodes with messages and queues

Learn how to use the MQInput, MQGet and MQOutput nodes to interact with queues defined on a local queue manager.

Import projects

When you click Import, a project will be created in your workspace: An application named ExampleMQ.

ExampleMQ provides two message flows named MQOutput and MQInput_MQGet.

The MQOutput message flow is driven by a FileInput node configured to receive an XML input message containing three famous couples (Romeo and Juliet, Lennon and McCartney, Laurel and Hardy). The FileInput is wired to a Compute node which loops over the data, dividing each pair. MQOutput nodes send separate messages (but both given the same MsgId) with the name of one person to QUEUE1 and the name of the other person to QUEUE2

The MQInput_MQGet message flow uses an MQInput node to read a message from QUEUE1. Next, an MQGet node reads the associated message (using the matching MsgId) from QUEUE2. Finally a Trace node records the re-unified information from the message pair to a local file.

Once imported using the link on the right, the application ExampleMQ is ready to be deployed and tested. To deploy, you can use a provided BAR file, as described on the Prepare tab.

Import

Prepare a trace directory, create a queue manager and integration node, and then deploy the message flows

The tutorial includes a Trace node which uses a trace directory. The tutorial also requires MQ definitions for two queues. Execute the following steps to prepare your environment accordingly:

  1. The message flow's Trace node will attempt to write a trace file to the directory C:\temp. Create this directory if it does not already exist, or edit the Trace node in the message flow to use a different directory suitable for your runtime environment. The tutorial scenario also uses a FileInput node which is configured to read from this directory, so if you choose an alternative you will also need to modify this part of the flow.
  2. Create an IBM MQ queue manager and configure it by completing the following steps:
    1. Open the IBM App Connect Enterprise Console. If you don't already have one, create and start a queue manager:
      crtmqm TESTQM
      strmqm TESTQM
    2. Next create the two queues by typing the following commands:
      runmqsc TESTQM
      define ql(QUEUE1)
      define ql(QUEUE2)
  3. Create an integration node which is associated with the above queue manager by completing the following steps:
    1. In the console window, create an integration node by typing the following command:
      mqsicreatebroker TESTNODE -q TESTQM
    2. Start the integration node by typing the following command:
      mqsistart TESTNODE
    3. Create an integration server associated with the integration node by typing the following command:
      mqsicreateexecutiongroup TESTNODE -e default
  4. Back in the Toolkit, drag and drop deploy the provided BAR file named ExampleMQ.bar which you will find inside the ExampleMQ application.

Use the Flow Exerciser to test the tutorial scenario

  1. Open MQOutput.msgflow.
    • Click the Flow Exerciser icon to start testing the flow (when challenged you do not need to redeploy the flow again).
    • Copy the input file named input.xml to the input directory which is being monitored by the FileInput node deployed as part of message flow MQOutput, which by default is C:\temp.
    • Click the View Path icon and a green line is plotted showing the path travelled by the message through the flow. You can click on the wires between nodes to view what the message data looked like at different points in the flow.
    • To inspect the result of the second message flow MQInput_MQGet.msgflow, using a text editor open the trace file C:\temp\TraceMQInput_MQGet.txt, which should contain three entries with correctly re-paired data like this:
